TWK Integration SDK

A ready‑to‑use Angular experience to integrate TWK capabilities (user data, theming, media, and more) into your services with a secure, mobile‑friendly UI.

Get Started View Documentation

What is this SDK?

This project bundles an Angular UI that talks to the TWK native bridge through the global TWK object (defined in assets/js/twkhelper.js). It handles:

  • Reading user and device information from TWK APIs
  • Auto‑detecting device language and layout direction (LTR / RTL)
  • Applying light / dark appearance using the TWK capabilities endpoint
  • Displaying transport documents and carrier cards in a responsive layout

Quick integration

  1. Host this Angular build in your web container or TWK webview.
  2. Include the TWK helper script before the Angular bundle:
<script src="assets/js/twkhelper.js"></script>
<script src="main.js" type="module"></script>
  1. Configure TWKAPIBASE, SHAREDSECRET, and ENABLELOG in your host.
  2. Open this module from your native or web shell as an embedded service.

SDK documentation

Environment & configuration

The SDK expects the following globals to be configured by the host environment before loading assets/js/twkhelper.js:

  • TWKAPIBASE – Base URL for the TWK bridge API.
  • SHAREDSECRET – Secret used to sign requests (HMAC‑SHA256).
  • ENABLELOG – Optional boolean to enable verbose logging.

Appearance & dark mode

The ThemeService calls TWK.getDeviceInfo() and automatically toggles a dark-mode class on the <body> element. Dark‑theme tokens are defined in assets/scss/_dark.scss.

The landing page itself uses a fully black background to match a dark introduction experience, while still respecting the global dark‑mode tokens for cards and text.

Key TWK helpers

The TWK helper exposes a set of methods on window.TWK, for example:

  • TWK.getUserFullName() – used to display the signed‑in user.
  • TWK.getDeviceInfo() – used for dark / light appearance.
  • TWK.getUserVehicles() – can be used to build transport flows.

You can extend this Angular SDK by injecting your own services that wrap these calls, or by adding new pages that consume the same helpers.
